# NLP_gpt2_poetry_experiment 可控文本生成与模型行为分析

## 一、方法简述

### 1.1 数据集
本实验使用自建的中文古诗数据集，共包含 50 首唐诗（五言诗 27 首，七言诗 23 首），原始格式为 `[T]标题` 后跟诗句。经过预处理，将每首诗构造成 `体裁：标题 诗句` 的形式，例如：
> 五言诗：静夜思 床前明月光，疑是地上霜。举头望明月，低头思故乡。

为进一步提升微调效果，采用数据增强策略：将 50 首诗重复 2 次并随机打乱，得到 **100 条训练样本**（文件 `train_poems_augmented.txt`）。

### 1.2 模型架构
采用 Hugging Face 提供的轻量级中文预训练模型 **`uer/gpt2-chinese-cluecorpussmall`**（约 100M 参数）。该模型基于 GPT‑2 架构，适用于 CPU 环境下的微调与生成实验。

### 1.3 核心训练参数
- 训练轮数：**8 轮**
- 批次大小：`2`  
- 学习率：`5e-5`  
- 最大序列长度：`128`  
- 优化器：AdamW  
- 热身步数：`50`  
- 保存步数：`100`  

训练过程中损失从 **3.90 下降至 0.15**，困惑度（Perplexity）从约 49.4 降至约 1.17，表明模型成功学习了古诗的词汇分布与句法特征。

### 1.4 生成参数设置
所有生成实验固定随机种子，对比以下策略：
- **采样策略**：贪婪搜索（`do_sample=False`）、束搜索（`num_beams=5`）、随机采样（`do_sample=True`）
- **温度参数**：`0.3`（低温）、`1.0`（中温）、`1.5`（高温）
- **Top‑p 采样**：`p=0.9`

## 三、深度分析与讨论

### 3.1 不同生成策略的对比

| 策略 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| 贪婪搜索 | 生成速度快，输出确定性高 | 容易陷入重复循环，缺乏多样性 | 需要唯一正确答案的任务（如问答） |
| 束搜索 | 句子更通顺，整体逻辑性较强 | 仍偏向高概率词，创造性不足 | 机器翻译、文本摘要 |
| 随机采样（中温） | 多样性好，能产生新颖组合 | 可能出现不连贯或离题内容 | 诗歌生成、故事创作 |
| Top‑p 采样 | 动态调整候选集，平衡质量与创意 | 参数 `p` 需要调优 | 通用文本生成 |

从实验结果看，贪婪搜索生成的“就是这样一个人”偏向口语化；束搜索生成了“最是一年春好处”“近乡情更怯”等名句拼接，虽通顺但缺乏新意；而随机采样在温度 1.0 时产生了“春来发几枝”与“葡萄美酒夜光杯”的混合，具有一定诗意但逻辑跳跃。

### 3.2 温度参数的影响

温度 τ 控制 softmax 输出概率的平滑度：
- **τ=0.3（低温）**：概率分布尖锐，模型几乎总是选择最高概率的词，导致输出与贪婪搜索类似，并出现无意义的重复（如“胡萝卜素”）。这是因为低温放大了训练数据中的高频模式，模型趋向于安全但僵化的生成。
- **τ=1.0（中温）**：原始概率分布，模型在合理与创意之间取得平衡，生成的“古来征战几人回”等诗句既有古诗韵味又不完全照搬原文。
- **τ=1.5（高温）**：概率分布平坦，低概率词被选中的机会增加，产生“车成泪痕相照两千里路”等陌生化表达，虽具创造性但常伴随逻辑断裂。

### 3.3 模型未能生成严格五言/七言格式的原因分析

尽管微调后模型能输出具有古诗风格的词汇（如“金谷园”“征人尽望乡”），但生成的句子长度不固定，无法自觉遵守“每句五字或七字”的硬性约束。原因如下：

1. **模型架构的本质限制**  
   GPT‑2 是一个**自回归字符（子词）预测模型**，它没有内置的“字数计数器”或“韵律规则”。每一步只根据前面的 token 预测下一个最可能的 token，其学习目标是最大化条件概率，而非满足符号化的格式要求。

2. **训练数据的表征方式**  
   我的训练样本中，诗句以连续字符串形式存储（例如“床前明月光，疑是地上霜。”）。模型从中学会的规律是“在某些位置出现逗号、句号是常见的”，但并未显式学习到“五个字符后必须出现逗号”的逻辑。如果将数据预处理成每句后加换行符 `\n`，模型可能学会换行模式，但仍难以精确控制字数。

3. **预训练知识的干扰**  
   基础模型在通用中文语料（含大量现代散文、新闻）上预训练，已经学得“自由字数”的表达习惯。微调数据量（100 条）相对不足，无法完全覆盖预训练中关于现代汉语的统计权重，因此模型仍会“回忆”起非古诗的句式（如“就是这样一个人”）。

4. **评价任务与训练目标的偏差**  
   语言模型训练目标是**最小化下一个 token 的交叉熵损失**，而不是“格式正确性”。只要生成的 token 序列在统计上与训练集相似，即使字数不符，损失也不会显著升高。因此模型更倾向于生成概率高的通用短语，而不是严格数着字输出。

### 3.4 体裁控制能力的局限

补充实验表明，微调后的模型无论输入“五言诗：”还是“七言诗：”，均倾向于生成七言诗（如“回乐烽前沙似雪”和“荷叶罗裙一色裁”）。原因在于：
- 训练数据中七言诗占比略低（23/50），模型未充分建立体裁与提示词的强关联。
- 提示词出现在序列开头，随着生成步数增加，其注意力权重被后续字符稀释。
- 实现可靠的体裁控制需要更大规模、更平衡的数据集，或使用条件生成模型（如加入特殊的控制 token）。

### 3.5 微调效果总结

虽然模型未能生成严格格式化的古诗，但微调后的输出质量相比微调前有显著提升：
- **词汇层面**：从“花样姐姐”“浪子回头”等现代词转向“征人尽望乡”“金谷园”“葡萄美酒夜光杯”等唐诗常见意象。
- **句法层面**：开始使用“不知何处吹芦管，一夜征人尽望乡”这类七言对仗结构，局部连贯性增强。
- **损失曲线**：从 3.90 降至 0.16，困惑度从 49.4 降至 1.17，定量指标证实模型确实“学进去了”。

### 3.6 对神经网络“创造性”的实证认识

本实验揭示了神经生成模型“创造性”的本质：**它并非真正的灵感或意图，而是对训练数据概率分布的一种随机采样与插值**。不同生成参数（温度、采样策略）相当于调节采样的“随机性强度”，从而在“复制”（低温/贪婪）与“幻想”（高温/随机）之间连续变化。模型无法主动遵循外部规则（如字数限制），除非这些规则被大量地、显式地编码到训练数据中。

---

## 总结

本项目通过微调轻量级中文 GPT‑2 模型并系统性调整生成参数，深入探究了采样策略与温度对文本生成质量、多样性的影响。实验发现：束搜索和中等温度能较好平衡流畅性与创造性；低温易导致重复，高温则引发逻辑断裂；模型难以自动遵守格式约束，反映出现有自回归架构在符号化规则理解上的本质局限。这些结论为理解神经网络的可控文本生成提供了实证依据。
